# Contributing Guide

このプロジェクトへの貢献ガイドラインです。

## コミットメッセージ規約

### フォーマット

```
<英語のタイトル>

<日本語の本文>
```

### ルール

1. **タイトル（1行目）**: 英語で記述
   - 50文字以内を推奨
   - 命令形を使用（例: "Add feature" not "Added feature"）
   - 最初の文字は大文字
   - 文末にピリオドを付けない

2. **本文（3行目以降）**: 日本語で記述
   - タイトルと本文の間に空行を入れる
   - 変更の理由や背景を説明
   - 何を変更したかではなく、なぜ変更したかに焦点を当てる

### 例

```
Update ScoreInput with winner-first workflow

スコア入力のUXを改善するため、ワークフローを変更：
- 最初に勝者を選択する大きなボタンを追加（プレイヤー1/プレイヤー2）
- その後、敗者のスコア（0-7点）のみを入力
- 勝者には自動的に14点が割り当てられる
- 次のラックに移動する際に入力状態をリセット

この変更により、「14」と入力する際に「1」で誤った自動入力が
発生する問題が解消され、より直感的な入力プロセスになります。
```

## 開発環境

```bash
# 依存関係のインストール
pnpm install

# 開発サーバーの起動
pnpm dev

# Lintとフォーマット
pnpm lint

# ビルド
pnpm build
```

## プルリクエスト

1. `main`ブランチから新しいブランチを作成
2. 変更を加えてコミット（上記のコミットメッセージ規約に従う）
3. プルリクエストを作成
4. レビューを受けてマージ
